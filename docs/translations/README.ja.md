# LRC Studio
 
曲の歌詞とオーディオを同期させるためのプロフェッショナルなWebアプリケーションです。カラオケスタイルの単語レベルのタイミング機能、多言語コンテンツ、クラウドプロジェクトなどをサポートしています。

## 機能

### 🎵 オーディオソース

- **ローカルファイル** — ドラッグ＆ドロップまたは参照してMP3、WAV、FLAC、OGGなどの一般的なフォーマットを読み込みます。シークやループのハンドルを備えた完全な波形表示をサポート。
- **YouTube** — 任意のYouTube URLを貼り付けて、エディタ内で直接オーディオをストリーミングします。A-Bループをサポートする視覚的な再生進捗インジケーターが含まれています。
- **Spotify** — OAuth経由でSpotify Premiumアカウントを接続し、エディタ内でSpotifyのカタログから直接トラックを読み込んで再生します。

### エディターモード

- **LRC モード** — ミリ秒精度のタイムスタンプによる行ごとの同期。設定可能なキー（デフォルト: `スペース`）を押して、アクティブな行に現在の再生位置をタイムスタンプとして記録します。
- **SRT モード** — 行ごとの開始・終了時間を持つ字幕フォーマット。正確な字幕配置のために、各行にインとアウトのタイムスタンプが含まれます。
- **Words モード** — カラオケのような単語レベルの同期。行内の各単語が独自のタイムスタンプを持ち、プレビューでのカラオケ塗りつぶしアニメーションを可能にします。

### タイミングと同期

- **キーボード駆動のタイムスタンプ** — 同期セッション中にタイムスタンプの設定、微調整、行のナビゲーションを行うための設定可能なショートカットキー。
- **微調整（ナッジング）** — 設定可能な増分（デフォルト: 0.1秒 / 0.01秒 (微細)）でタイムスタンプを前後にシフトします。
- **一括シフト** — 複数の行を選択して、それらのタイムスタンプを同時にシフトします。
- **グローバルオフセット** — すべてのタイムスタンプを一度にシフトします。
- **A-Bループ** — 波形またはプログレスバーでループの始点と終点を設定し、特定の部分を繰り返し練習できます。
- **タイムスタンプ後の自動一時停止** — タイムスタンプを設定した後、オプションで自動的に再生を一時停止します。
- **タイムスタンプの重複検出** — 2つの行が同じタイムスタンプを共有している場合、視覚的なバッジで警告します。

### 歌詞コンテンツ

- **セカンダリ歌詞** — 行ごとにサブのテキストトラック（例：ローマ字、他の言語の読み方など）を追加します。
- **翻訳レイヤー** — 各行に別々の翻訳フィールド（例：日本語の歌詞と並べた英語の翻訳）を持たせます。
- **ふりがな / ルビのマークアップ** — `{字|じ}`の構文を使用して、CJK文字に読み方の注釈を付けます。エディタは文字の上にルビをインラインでレンダリングします。
- **CJK対応のトークン化** — 日本語、中国語、韓国語のテキストは自動的に文字ごとに分割され、文字単位のタイミングを設定できます。ラテン語などの文字列は単語全体として維持されます。
- **読み方フォーマット** — 発音のルビ表示をひらがなとカタカナで切り替えます。
- **2行表示** — 読みやすさを向上させるために、同期中にアクティブな行の下に次の行をオプションで表示します。

### インポート

- **貼り付け / 入力** — エディタのテキスト入力欄に直接歌詞を入力します。
- **ファイルのインポート** — 既存の`.lrc`、`.srt`、または`.txt`ファイルを読み込みます。単語レベルのLRC（拡張LRC）もサポートされており、単語ごとのタイムスタンプが保持されます。
- **URLインポート** — インポートパネルを使用して、リモートURLから歌詞をインポートします。
- **貼り付けの検出** — エディタにLRCまたはSRTブロックを貼り付けると、自動的に検出して解析されます。

### エクスポート

- **LRCのダウンロード** — オプションの単語タイムスタンプ（`<MM:SS.xx>`）を含む標準的な`[MM:SS.xx]`フォーマット。
- **SRTのダウンロード** — 2行目のセカンダリコンテンツをオプションで含む`HH:MM:SS,ms --> HH:MM:SS,ms`フォーマット。
- **クリップボードへのコピー** — コンパイルされたLRCまたはSRTの出力を瞬時にコピーします。
- **LRCメタデータタグ** — オプションで`[ti:]`、`[ar:]`、`[al:]`、`[lg:]`のヘッダータグを含めることができます。
- **設定可能な精度** — 行および単語のタイムスタンプについて、小数点以下2桁（`[01:23.45]`）または3桁（`[01:23.456]`）の精度を選択できます。
- **改行コード** — 異なるメディアプレーヤーとの互換性のために、LFまたはCRLFを選択します。
- **空行の削除** — 出力から未同期または空白の行を自動的に除外します。
- **タイムスタンプの正規化** — エクスポート時にタイムスタンプを並べ替え、重複を削除します。
- **ファイル名パターン** — `lyrics.lrc`（固定）またはプロジェクトのタイトルに基づくファイル名でエクスポートします。
- **翻訳とセカンダリの切り替え** — エクスポートされるファイルにセカンダリトラックや翻訳を含めるかどうかを選択します。
- **サーバーコンパイルとローカルフォールバック** — エクスポートは高度なフォーマット処理のためにサーバー上でコンパイルされます。サーバーが利用できない場合は、自動的にローカルコンパイラにフォールバックします。

### ライブプレビュー

- **リアルタイムカラオケプレビュー** — プレビューパネルは現在再生中の行を強調表示し、再生に合わせて文字ごとに単語を塗りつぶします（Wordsモード時）。
- **カラオケ塗りつぶしイージング** — 単語の塗りつぶしアニメーションを線形（正確）またはイーズイン/アウト（滑らか）から選択します。
- **塗りつぶしトラックの選択** — アニメーションによる塗りつぶしをメインテキスト、セカンダリテキスト、またはその両方に適用します。
- **翻訳の表示** — プレビューで各歌詞の行の下にオプションで翻訳レイヤーを表示します。
- **ふりがなの表示** — プレビューでのルビテキストの注釈の表示を切り替えます。
- **フォントサイズ** — 小、標準、大、特大の4つのサイズプリセットがあります。
- **行間隔** — 行間の間隔を狭く、標準、広くから選択できます。
- **プレビューの配置** — 左揃え、中央揃え、右揃えから選択できます。
- **自動スクロール** — アクティブな行が自動的に表示領域にスクロールされます。配置（中央、上部、最も近い、オフ）とスクロールの動作（スムーズ / 即時）を設定可能です。

### プロジェクト管理

- **プロジェクトライブラリ** — すべてのプロジェクトを参照、検索、管理します。各プロジェクトには歌詞、タイムスタンプ、メディア参照、およびエディタの状態が保存されます。
- **クラウド同期** — 認証されたユーザーのプロジェクトはサーバーに自動的に保存されます。プロジェクトは初回の保存時に作成され、効率のために差分のみパッチ適用されます。
- **自動保存** — 設定可能な時間間隔が経過するか、5回の行の編集が行われた後のいずれか早い方で、自動保存がトリガーされます。
- **手動保存** — 自動保存のステータスインジケーター（スピナー → チェックマーク）付きの保存ボタン。
- **ローカルストレージへのフォールバック** — オフラインでも作業が失われないように、すべてのプロジェクトデータは`localStorage`にもミラーリングされます。
- **アップロードライブラリ** — 過去にプロジェクトに関連付けられたすべてのオーディオファイルとYouTube/Spotifyトラックを表示します。
- **共有プロジェクトビューアー** — 埋め込まれたメディアと同期された歌詞を含む共有プロジェクトの読み取り専用ビューを開きます。
- **プロジェクトのメタデータ** — プロジェクト名、説明、タグを保存します。

### 共有

- **公開/非公開の共有** — 任意のプロジェクトの共有可能なURLを生成します。可視性を公開（リンクを知っている人全員）と非公開（所有者のみ）の間で切り替えます。
- **タイムスタンプ付きのディープリンク** — 特定の秒数から再生を開始するための`?s=N`クエリパラメータを共有URLに含めます。
- **現在の位置ショートカット** — 共有用のタイムスタンプを現在の再生位置にワンクリックで同期します。

### インターフェースとテーマ

- **テーマ** — ダーク、ライト、Dracula、Alucard、Alucard Light、システム（OSの設定に合わせる）。
- **アクティブ行のハイライトスタイル** — グロー、ズーム、カラー、またはディム。
- **翻訳のレイアウト** — スタック（歌詞の下に翻訳）またはサイドバイサイド。
- **フォーカスモード** — エディタパネルを非表示にしてプレビューを大きく表示します。
- **サイズ変更可能なパネル** — エディターとプレビューの間の区切り線をドラッグして幅を調整します。
- **レイアウトのロック** — パネルの偶発的なサイズ変更を防ぎます。
- **モバイルレイアウト** — プレーヤー、エディター、プレビューパネルをタブでナビゲートします。
- **国際化** — 完全なUIが**英語**と**スペイン語**で利用可能です（`/locales`フォルダーを介してさらに追加可能）。

### 設定パネル

アプリケーションは、設定モーダルを介して高度にカスタマイズ可能です。設定は論理的なパネルにグループ化されています：

**エディター**
- **デフォルトのエディターモード**: LRCまたはSRT。
- **単語のタイムスタンプの表示**: エディタで単語レベルの`<MM:SS.xx>`タイムスタンプの表示を切り替えます。
- **次の行の自動選択**: スタンプ後に自動的に選択を次の行に進めます。
- **ハイライトモード**: アクティブな行の強調表示方法を制御します。
- **選択範囲のループ**: 選択された行で囲まれたオーディオを自動的にループ再生します。
- **マーク時に一時停止**: 行をスタンプした直後に自動的に再生を一時停止します。
- **シフトの設定**: 「すべてシフト」アクションで適用される時間、およびデフォルトのナッジ量（例：0.1秒）を設定します。

**再生**
- **シーク時間**: シーク時に進む/戻る秒数（デフォルトは5秒）。
- **デフォルトの速度**: 再生速度の乗数（0.5倍から1.5倍、またはカスタム）を設定します。
- **音量コントロール**: ミュート切り替え付きの常に表示される音量スライダー。

**インターフェース**
- **テーマ**: ダーク、ライト、Dracula、Alucard、Alucard Light、またはシステム。
- **言語**: 英語またはスペイン語。
- **アクティブ行のハイライト**: グロー、ズーム、カラー、またはディム。
- **スクロール動作**: スムーズまたは即時のスクロール。
- **スクロール配置**: 中央、最も近い、上部、またはオフ。
- **プレビュー配置**: 左、中央、または右。
- **タイポグラフィ**: フォントサイズ（小から特大）および行間（コンパクトからリラックス）を調整します。
- **翻訳のレイアウト**: サイドバイサイドまたはスタック表示。
- **カラオケの塗りつぶし**: 塗りつぶしの対象（メイン、セカンダリ、または両方）とイージング関数（線形またはイーズイン・アウト）をカスタマイズします。
- **読み方フォーマット**: ルビの注釈をひらがなまたはカタカナで表示します。
- **レイアウトのロック**: エディタパネルの誤ったサイズ変更を防ぎます。

**エクスポート**
- **フォーマット**: コピーおよびダウンロードのデフォルトフォーマット（LRCまたはSRT）。
- **タイムスタンプ精度**: 行および単語レベルのタイムスタンプについて、小数点以下2桁（デフォルト）または3桁。
- **ファイル名パターン**: 固定（`lyrics.lrc`）またはメディアタイトルから派生。
- **フォーマットオプション**: Windows（CRLF）またはUnix（LF）の改行、空行の削除、タイムスタンプの正規化、メタデータタグの追加。

**ショートカット**
- エディター、プレーヤー、およびプレビューにわたる18のキーバインドとマウス修飾キーを設定します。競合検出機能が組み込まれています。

**高度な設定**
- **自動保存**: 自動保存の切り替えと間隔の設定（10〜300秒）。
- **タイムゾーン**: サーバーのスナップショットに使用されるタイムゾーンの自動検出または手動設定。

**接続とプロフィール**
- Spotify OAuthリンクの管理、表示名/アバターの更新、アカウントのパスワード変更。

## はじめに

### Dockerの使用（推奨される簡単なセットアップ）

> [!IMPORTANT]
> コマンドを実行する前に、**Docker Desktopが起動していることを確認してください**。Windowsを使用している場合は、Dockerエンジンが完全に初期化されていることを確認してください。

DockerとDocker Composeがインストールされている場合は、次の1つのコマンドでスタック全体を起動できます：

```bash
docker-compose up -d --build
```

これにより、以下のことが行われます：
1. クライアントのビルド（Nginx + 静的ファイル）
2. サーバーのビルド（Node.js）
3. MongoDBインスタンスの起動
4. すべてを相互に接続

アプリケーションは [http://localhost](http://localhost) で利用可能になります。

---

### 手動インストール

### 前提条件

- Node.js v18 以上
- npm

### インストール

1. リポジトリをクローンします:

   ```bash
   git clone https://github.com/crimsonCarnival/lrc-studio.git
   cd lrc-studio
   ```

2. クライアントとサーバーの両方の依存関係をインストールします:

   ```bash
   # クライアントの依存関係をインストール
   cd client
   npm install

   # 別のターミナルで、サーバーの依存関係をインストール
   cd ../server
   npm install
   ```

3. 環境変数を設定します:

   **クライアント (`client/.env`):**
   ```bash
   cp .env.example .env
   # ローカル開発用に VITE_API_URL が http://localhost:3000 に設定されていることを確認してください
   ```

   **サーバー (`server/.env`):**
   ```bash
   cp .env.example .env
   # MONGODB_URI およびその他の必要なキーを設定してください
   ```

4. 開発サーバーを起動します:

   **サーバー:**
   ```bash
   cd server
   npm run dev
   ```

   **クライアント:**
   ```bash
   cd client
   npm run dev
   ```

5. ブラウザで [http://127.0.0.1:5173](http://127.0.0.1:5173) を開きます。

### 利用可能なスクリプト

| コマンド | 説明 |
|---|---|
| `npm run dev` | Vite開発サーバーを起動します |
| `npm run build` | 本番用バンドルをビルドします |
| `npm run preview` | ローカルで本番ビルドをプレビューします |
| `npm run lint` | ESLint を実行します |

## LRC ファイル形式のリファレンス

### 標準 LRC

```lrc
[ti:Song Title]
[ar:Artist Name]
```

### セカンダリ / ふりがな 付き

互換性のあるプレーヤーでは、タイムスタンプを共有する複数の行が縦に重ねて表示されます：

```lrc
[00:10.50]持ち上げて
[00:10.50]mochiagete
[00:10.50]Lift me up
```
